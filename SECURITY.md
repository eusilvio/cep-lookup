# Security Policy

## Supported versions

Only actively maintained versions receive security patches. See `SUPPORT.md`
for current maintenance windows.

## Reporting a vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report vulnerabilities privately using GitHub Security Advisories:

1. Open the repository on GitHub.
2. Go to the `Security` tab.
3. Click `Report a vulnerability`.
4. Submit the details through the private advisory form.

Include as much detail as possible:

- Affected package and version.
- Reproduction steps or proof of concept.
- Potential impact.
- Any suggested remediation.

You will receive an acknowledgment within 72 hours.
